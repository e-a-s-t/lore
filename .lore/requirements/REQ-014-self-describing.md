# REQ - Self Describing

## Requirement

Lore SHALL be developed using Lore itself.

## Rationale

The project should prove the model by storing its own requirements, ADRs, stories and tests in `.lore/`.

## Acceptance Criteria

- [ ] The Lore repository contains `.lore/requirements`.
- [ ] The Lore repository contains `.lore/adrs`.
- [ ] The Lore repository contains `.lore/stories`.
- [ ] The Lore repository contains `.lore/tests`.
